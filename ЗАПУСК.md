# Studance — Инструкция по запуску

## Требования

- **Node.js** версии **≥ 22.5.0** — [nodejs.org](https://nodejs.org)
- **npm** (поставляется вместе с Node.js)

Проверить версию: `node -v`

---

## Быстрый старт

```bash
# 1. Перейти в папку проекта
cd Studance

# 2. Установить зависимости (если node_modules отсутствует)
npm install

# 3. Запустить в режиме разработки
npm run dev
```

Сайт откроется на **http://localhost:4321**

---

## Команды

| Команда | Что делает |
|---------|-----------|
| `npm run dev` | Запускает dev-сервер с горячей перезагрузкой |
| `npm run build` | Собирает production-сборку в папку `dist/` |
| `npm run preview` | Превью production-сборки локально |

## Переменные окружения

Создайте файл `.env` в корне проекта (рядом с `package.json`):

```env
# Пароль для доступа к админ-панели /admin
ADMIN_TOKEN=ваш_секретный_пароль
```

Без `ADMIN_TOKEN` административная панель недоступна.

## Управление контентом

Данные сайта хранятся в `data/cms.json`. Редактировать можно:

- **Вручную** — открыть файл и изменить JSON
- **Через админ-панель** — перейти на `/admin`, ввести `ADMIN_TOKEN` из `.env`

### Добавление фото преподавателей

В `data/cms.json` у преподавателя укажите поле `photoUrl`:

```json
{
  "id": "t1",
  "name": "Анна Петрова",
  "role": "Бачата, Сальса",
  "experience": "8 лет",
  "photoUrl": "/photos/anna.jpg"
}
```

Файл фото положите в папку `public/photos/`.

## Production-запуск

```bash
# Собрать
npm run build

# Запустить сервер (порт 4321 по умолчанию)
node dist/server/entry.mjs
```

Для смены порта:

```bash
PORT=3000 node dist/server/entry.mjs
```

## Частые вопросы

**Q: После `npm run dev` браузер ничего не открыл автоматически?**  
A: Откройте вручную: http://localhost:4321

**Q: Ошибка `EADDRINUSE`?**  
A: Порт 4321 занят. Укажите другой: `npm run dev -- --port 3000`

**Q: Форма записи не отправляется?**  
A: Сервер должен быть запущен (`npm run dev` или production-режим). Статический preview (`npm run preview`) не поддерживает API.
